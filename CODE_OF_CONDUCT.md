# Code of Conduct

## @mookielianhd/n8n-nodes-instagram

This project is committed to providing a respectful, professional, and inclusive environment for all contributors and users. By participating in this repository, you agree to follow this Code of Conduct.

---

## 1. Purpose

The purpose of this Code of Conduct is to:

- Promote respectful collaboration
- Ensure responsible and lawful use of this package
- Maintain a constructive and inclusive community
- Protect users, contributors, and maintainers

This Code of Conduct applies to all interactions in:

- GitHub issues
- Pull requests
- Discussions
- Documentation
- Any related community spaces

---

## 2. Expected Behavior

Participants are expected to:

- Communicate respectfully and professionally
- Provide constructive feedback
- Accept feedback gracefully
- Focus on technical issues rather than individuals
- Help maintain a welcoming environment for contributors of all skill levels

---

## 3. Unacceptable Behavior

The following behaviors are not tolerated:

- Harassment, discrimination, or hate speech
- Personal attacks or inflammatory language
- Trolling, insulting, or demeaning comments
- Publishing private or sensitive information
- Spamming issues or pull requests
- Deliberately misusing the project to cause harm

---

## 4. Responsible Use of the Package

This package integrates with Instagram through official APIs. Users must:

- Comply with all applicable laws and regulations
- Follow Meta/Instagram Platform Terms and API policies
- Respect user privacy and data ownership
- Avoid spam, abusive automation, or unauthorized messaging
- Not attempt to bypass API restrictions, rate limits, or safeguards

The maintainers are not responsible for misuse of this package.

---

## 5. Security Practices

- Never share access tokens, secrets, or credentials publicly.
- Do not include sensitive data in issues, logs, or pull requests.
- If a security vulnerability is discovered, report it privately to the maintainer.
- Allow time for responsible disclosure before public discussion.

---

## 6. Reporting Issues

When reporting bugs:

- Provide clear reproduction steps
- Include expected vs actual behavior
- Specify environment details (n8n version, Node version, OS)
- Include relevant logs (without sensitive data)

Before opening a new issue:

- Search existing issues to avoid duplicates
- Ensure the issue is relevant to this package

---

## 7. Contributions

Contributors should:

- Follow the existing code style and structure
- Write clear commit messages
- Avoid introducing breaking changes without discussion
- Document new features or configuration changes
- Ensure changes do not expose credentials or sensitive data

Maintainers reserve the right to decline contributions that do not align with the projectâ€™s goals or standards.

---

## 8. Enforcement

Maintainers may take action if this Code of Conduct is violated, including:

- Requesting edits or removal of content
- Closing issues or pull requests
- Restricting participation
- Blocking users in severe cases

Decisions are made at the discretion of the maintainers to protect the project and its community.

---

## 9. Scope

This Code of Conduct applies only to this repository and related project spaces. It does not govern external communities or third-party platforms.

---

By participating in this project, you agree to abide by this Code of Conduct.
